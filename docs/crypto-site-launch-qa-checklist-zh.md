# 加密网站上线 QA 检查表

本文档亦提供[英文版](./crypto-site-launch-qa-checklist.md)。

## 适用场景

这份检查表适用于第一次公开上线、一次性发布 5 篇及以上页面、模板发生改动之后，以及返佣模块或结构化数据上线之后。

## 发布闸门

- [ ] 本批次上线清单已经冻结。
- [ ] 工程、内容、审核、监控负责人已经明确。
- [ ] 首页、文章模板、对比页模板、工具页模板和信任页都在检查范围内。
- [ ] 已经定义回滚负责人和回滚触发条件。

## 信任与政策检查

- [ ] 隐私政策已上线。
- [ ] 服务条款已上线。
- [ ] 编辑政策已上线。
- [ ] 更正与更新政策已上线。
- [ ] 联系页至少提供两种联系方式。
- [ ] 市场风险免责声明已上线。
- [ ] 联盟披露语位于第一处商业化链接之前或同一屏可见位置。
- [ ] 联盟链接带有 `rel="sponsored nofollow"`。
- [ ] 所有商业页面都显示最近核验日期和核验人信息。

## Staging / Preview 检查

- [ ] 生产环境的 `robots.txt` 已检查，未错误屏蔽重要路径。
- [ ] `sitemap.xml` 存在，且只包含可收录 URL。
- [ ] 重要模板没有带着 `noindex` 上线。
- [ ] 首页、文章、对比、工具、信任页模板的 canonical 都正确。
- [ ] 每类模板都具备标题、meta description、`H1` 和结构化数据。
- [ ] 面包屑和内链渲染正常。
- [ ] 代表性页面已完成移动端检查。
- [ ] 每类模板都检查了 Core Web Vitals。
- [ ] 代表性页面的结构化数据验证通过。
- [ ] 披露模块和风险提示模块首屏渲染正常。

## 内容与证据检查

- [ ] 所有高波动事实声明都在证据台账中有对应记录。
- [ ] 截图与当前真实产品流程一致。
- [ ] 手续费、返佣、奖励、KYC、提币规则都有当前核验时间。
- [ ] 对比表和工具页输出与已核验声明一致。
- [ ] 页面上的最近核验日期与证据台账一致。

## 实体与品牌检查

- [ ] About 页面使用的品牌描述与 CMS 默认信息、结构化数据一致。
- [ ] 首发作者都具备作者页面。
- [ ] Organization schema 已部署且信息最新。
- [ ] 需要作者身份的页面具备 Person schema。
- [ ] Logo、品牌名、社交链接在关键页面一致。
- [ ] 如果品牌容易和其他实体混淆，已添加区分说明。

## 上线当天检查

- [ ] 生产部署完成且没有模板回归。
- [ ] 生产环境不存在误封禁指令。
- [ ] sitemap 已提交或重新提交。
- [ ] 部署后已人工抽检代表性 URL。
- [ ] 上线最初数小时内已监控 `404` 和 `5xx`。
- [ ] CTA 链接、返佣链接和工具表单已人工测试。
- [ ] 披露语、免责声明和核验元信息在生产环境渲染正常。

## 上线后检查

### T+1

- [ ] 抽查代表性 URL 的可收录状态。
- [ ] 检查抓取错误和服务器错误。
- [ ] 确认页面 canonical 和结构化数据正常。
- [ ] 确认排名、流量、Core Web Vitals 和安全告警已经启用。

### T+7

- [ ] 复查曝光、点击和初始关键词波动。
- [ ] 复查工具页 CTA 点击和返佣链接点击。
- [ ] 复查核心商业页的披露语、新鲜度和截图一致性。
- [ ] 复查所有 `review-needed` 状态的证据记录。

### T+30

- [ ] 按批次复查收录覆盖和 URL 健康度。
- [ ] 按模板类型复查页面表现。
- [ ] 对声明、截图或搜索匹配度已过期的页面执行刷新或重写。
- [ ] 判断这一批次是否适合进入下一波扩张。

## 回滚触发条件

- [ ] 重要页面被 `robots.txt` 或 `noindex` 阻断。
- [ ] canonical 指向错误 URL。
- [ ] 披露语或风险提示模块未渲染。
- [ ] 商业模板的结构化数据丢失。
- [ ] `404` 或 `5xx` 超过约定阈值。
- [ ] 关键 CTA、返佣跳转或工具流程在生产环境失败。

## 推荐 Skill 顺序

1. 在 staging 或生产域名上运行 `/seo:check-technical`。
2. 对首页和核心商业页运行 `/seo:audit-page`。
3. 当模板 schema 缺失或失效时，运行 `/seo:generate-schema`。
4. 当品牌、作者或组织信息出现漂移时，运行 `entity-optimizer`。
5. 在上线关闭前运行 `/seo:setup-alert`。
6. 在 T+7 和 T+30 运行 `/seo:report`。
