# SEO Image Placeholder Reference

Use this reference when inserting placeholder blocks into drafts.

## Placement Heuristics

Prefer placeholders in sections that need one of these jobs:

| Situation | Preferred placeholder | What to show |
|-----------|-----------------------|--------------|
| Brand or site authenticity matters | **Official domain + first screen** | Browser address bar, favicon/brand mark, hero section |
| A workflow step could feel abstract | **UI step proof** | Exact screen, button, menu path, or setting state |
| A claim needs visible evidence | **Result or metric proof** | Dashboard number, report slice, before/after state |
| A comparison needs grounding | **Side-by-side proof** | Two pages, plans, features, or UI states being compared |
| A quote/source needs context | **Source proof** | Headline, publication, author/byline, date, or cited section |

## Standard Block Format

Keep each placeholder in this exact three-line blockquote pattern:

> [截图占位符：一句话标题]
> 建议截图内容：明确写出页面、UI 元素、地址栏、数据或状态。
> 截图目的：说明这张图要降低什么理解成本，或增强什么信任信号。

## Template Library

### 1. Official domain trust

> [截图占位符：官方主域名与页面首屏]
> 建议截图内容：浏览器地址栏中的 `example.com`，以及页面首屏的品牌标识。
> 截图目的：把“这是官方站点”从文字判断变成可见证据。

### 2. Product workflow step

> [截图占位符：完成第 2 步时的设置界面]
> 建议截图内容：设置页左侧导航、当前选中的选项卡，以及保存成功后的状态提示。
> 截图目的：让读者快速确认自己是否来到正确页面并完成正确操作。

### 3. Evidence or result proof

> [截图占位符：结果页中的关键指标]
> 建议截图内容：结果页顶部摘要卡片，以及文中提到的核心指标或变化趋势。
> 截图目的：给“效果已经出现”提供直接可见的验证点。

## Self-Check

Before returning the draft, verify:

1. Every placeholder is attached to a specific section or step.
2. The screenshot brief names concrete visual elements, not vague themes.
3. The purpose explains why the image improves comprehension, trust, or proof.
4. Decorative-only image slots are excluded unless the user explicitly asked for them.
5. The draft remains readable even before the screenshots are filled in.
